Your spoke VNet contains a server that represents a workload in the cloud. Until the VNet is peered to the vWAN hub, it is completely isolated. In this challenge you will connect it.

## Spoke details

<EnvVar field="spoke_cidr" label="Spoke VNet address space" />
<EnvVar field="spoke_server_private" label="Spoke server private IP" />

The diagram on your **My Environment** page will update to show the peering connection once it is established.

## Steps

### 1. Open the Virtual Hub

In the Azure Portal, navigate to:

**Virtual WANs → Connectivity → Virtual network connections**

### 2. Click "Add connection"

### 3. Select your hub, your resource group and your spoke vNet.

| Parameter | Value |
|-----------|-------|
| Connection name | `spoke<EnvVarInline field="env_id" />-connection` (or any other name that works for you) |
| Hubs | your hub |
| Subscription | your subscription |
| Resource Group | your resource group (vwanlab-student-XX)
| Virtual network | `spoke<EnvVarInline field="env_id" />Vnet` |
| Propagate default route | No |
| Associate Route Table | Default |

![](vnet-connection.png)

Click **Create** and wait for the connection to show **Succeeded** (typically 1–3 minutes).

### 3. Verify peering state

The **My Environment** page diagram should now show the spoke connected to the hub.
