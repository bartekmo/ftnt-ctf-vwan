## Overview

As the SD-WAN template focuses on a use case where branch devices build multiple tunnels to a single hub rather than single tunnels to multiple hubs you need to modify slightly the SD-WAN template for branch devices.

### SD-WAN branch template

1. Open the branch SD-WAN template you created earlier and linked to SD-WAN Overlay template.
1. move tunnels to both HUBs to the same SD-WAN zone
1. remove WAN1 and port1 from the template

### Configure SD-WAN rule

1. create a new SD-WAN rule for traffic from your branch network to your Azure spoke network (create new address objects)
1. Select the ***Lowest cost*** strategy and add both tunnels to the rule
1. Add the zone and optionally healthchecks
