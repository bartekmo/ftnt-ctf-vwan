## Overview

In this task you will configure an SD-WAN overlay template connecting branch offices (you have only one at the moment) with FortiGates in vWAN hub.

### Before you start

Before you create the template prepare a group for all branch office FortiGates:

1. in your ADOM go to **Device Manager** > **Device & Groups**
1. create a new group called ***branches***
1. add your branch FortiGate to it

## Create the template

Your SD-WAN network was already planned for you:
<EnvVar label="Loopback address space" field="sdwan_healthcheck_range" />
<EnvVar label="ASN" field="fgt_asn" />

In **Provisioning Templates**:

1. create a new dual-hub (active-active) **SD-WAN Overlay** template (note - it's not the same as SD-WAN template), **Name it after yourhub name** (eg. "hub42")
1. provide your regional loopback address space and ASN in ***Advanced*** section, enable *BGP on Loopback*, disable ADVPN
1. select each of your hubs as a ***Primary HUB***, select **branches** group as your ***Device Group 1 Assignment*** and enable automatic branch ID assignment
1. for each *Primary HUB* 
    - type in "port1" as *WAN Underlay 1*, 
    - enable **Override IP**, 
    - type in this instance public IP, 
    - delete *WAN Underlay 2*, 
    - change Network Advertisement to **Static**
1. in Branch section:
    - type in "port1" for *WAN Underlay 1*
    - delete *WAN Underlay 2*
    - set *Network Advertisement* to **Connected** and select **port2**
1. enable ***Add Overlay objects (...)*** for both Branch and Hub templates, add *Overlay Interfaces and Zones* and *Health Check Servers*
